
Задача №1. (O(n²))

Цель: Реализация задачи Иосифа Флавия — определение последнего оставшегося элемента после циклического удаления каждого k-го элемента.

Параметры:
nums - Вектор чисел (участники круга).
k - Шаг, через который удаляются элементы.

Если в круге только один элемент, он и является решением.

Корректировка k:
Если k больше текущего размера очереди, используется остаток от деления (k = k_buf % deque.size()).

Удаление элемента:
Первые k-1 элементов перемещаются в конец очереди.
k-й элемент удаляется.

Если k = 0, удаляется последний элемент (нестандартное поведение для классической задачи).


Задача №2. (O(log n))

Структура класса EventScheduler:
events - Карта для хранения событий, где ключ — время, значение — название события.
priority_queue - Очередь с приоритетом, которая хранит времена в порядке возрастания (благодаря std::greater<>).

Метод add_event:
Если время time уже существует, событие перезаписывается в events.
Если времени нет, оно добавляется в events и priority_queue.

Метод process_next:
Извлекает ближайшее по времени событие.
Удаляет время из очереди и событие из карты.


Задача №3. (O(n log n))

Задача функции longest_duplicate_subarray:
Функция ищет максимальную длину повторяющегося подмассива в заданном векторе nums. 

Например:
Для {1,2,3,1,2,3} повторяющийся подмассив длины 3: [1,2,3].
Для {5,5,5,5} повторяющийся подмассив длины 3: [5,5,5].

Бинарный поиск для оптимизации:
low и high задают диапазон длин подмассивов для проверки.
Идея: 
Проверить, существует ли повторяющийся подмассив длины mid = (low + high) / 2. 
Если да — искать более длинные подмассивы, иначе — более короткие.

Алгоритм проверки для длины mid:
Генерация строкового представления каждого подмассива длины mid (например, "1,2,3,").
Проверка, встречался ли такой подмассив ранее (через unordered_set).
Если дубликат найден — прервать проверку.

Корректировка диапазона бинарного поиска:
Если дубликат найден — обновить max_length и продолжить поиск для больших длин.
Если нет — уменьшить длину.
